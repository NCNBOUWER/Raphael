# Cross-disciplinary manifest

Purpose: define how GeoMatrices extends from probability and classical architecture into cross-disciplinary ToE-candidate analysis.

## Domain model

A domain can be mapped by GeoMatrices when it supplies:

1. a finite or normalisable set of components,
2. a closure or conservation rule,
3. a coordinate or state-history sequence,
4. a measurable residual or failure condition,
5. a reason the mapped result matters.

## Candidate domains

| Domain | GeoMatrices representation | What it can do | What still requires external validation |
|---|---|---|---|
| CMB / cosmic asymmetry | irregular boundary map over a reference geometry | represent perturbation geometry and residual fields | empirical CMB residual tests and expert cosmology review |
| Biological ATGC / carcinisation | compositional trajectory through genotype/phenotype feature space | model repeated convergence as geometric trajectory | biological datasets, phylogenetic controls, evolutionary expert review |
| Macro-social / fiscal systems | closed fiscal or demographic simplex | compare allocation drift, centrality, and observer/factor effects | source-data stability, economic methodology review |
| Quantum/non-locality visualisation | MMB face/internal projection | visualise non-local aggregate vs local observer insufficiency | formal Bell-test or quantum-information validation |
| Dark-matter/large-scale structure | irregular density tensor/cosmic filament map | represent non-isotropic density as geometric deformation | astrophysical data and model comparison |
| Data systems/compression | closure-preserving state coordinates | compact state representation and anomaly detection | software benchmarks and independent implementation |

## ToE candidate language

GeoMatrices should be described as a **scale-free ToE candidate framework and falsification architecture for closed systems**. Its strongest internal claim is not that every domain is already empirically proven; its strongest claim is that the same closure, geometry, transport, observer, and falsification grammar can be applied across domains and made testable.

## Relation to EML#F

EML#F is not embedded here. Where EML-style binary derivation or catalogue language is required, this module should point outward to the EML#F folder once built. A GeoMatrices component may later exist inside EML#F, not the reverse.
