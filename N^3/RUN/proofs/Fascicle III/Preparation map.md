# Fascicle III preparation map

Scope: **Fascicle III — Raphael integration layer**.

Fascicle III is not another probability-only build and not another classical-geometry build. It is the integration layer that uses Fascicle I and II to organise, support, constrain, test, or disprove the fixed Raphael equation set.

## Fascicle stack

| Fascicle | Role | Output |
|---|---|---|
| I | mathematics and probability | TPW, product PMF, modal selector, Gamma bridge |
| II | classical analysis and architectural geometry | OBW, MMB, covariant bridge, irregular geometry |
| III | Raphael integration | equation crosswalk, support/disproof register, Raphael Suite hooks |

## Import rules

1. Import Fascicle I finite proofs as already established.
2. Import Fascicle II architectural proofs as already established.
3. Import Raphael equations as source-governed objects, not editable targets.
4. Import EML#F only as a separate third-party derivation/library source where relevant.
5. Keep workbook updates in place rather than producing new uncontrolled workbook versions.

| Output | Purpose | Status |

| Raphael claim crosswalk | identify support/constrain/disprove relationship per claim | pending |

## Proof posture

Fascicle III may classify a Raphael-facing proposition as:

- exact inheritance from Fascicle I,
- exact inheritance from Fascicle II,
- structural support,
- constraint condition,
- falsification test,
- empirical requirement,
- source-governed Raphael assertion,
- deferred to Raphael Lite / Raphael_Physics.

## Hard exclusions before build

- Do not write new Raphael equations.
- Do not rename Raphael equations to fit N^3.
- Do not expose private equations unless explicitly cleared.
- Do not merge EML#F into N^3 as if it were native.
- Do not build Raphael Lite yet.
- Do not publish ToE claims as proven without the required proof/status register.

## Build-lock questions

The owner must answer or accept defaults for equation exposure, source files, claim strength, public/private boundaries, and workbook sync before execution.
