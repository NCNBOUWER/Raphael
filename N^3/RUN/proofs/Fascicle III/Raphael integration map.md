## Integration principle

A Raphael-facing claim is stronger when it can be decomposed into:

1. a closure component from Fascicle I,
2. a bounded geometry component from Fascicle II,
3. a falsifiable domain prediction or measurable interface in Fascicle III,
4. an external empirical or expert-corroboration pathway.

This is the scientific-disproof stance: the framework is useful precisely because every elevated claim must expose how it would fail.

## Claim-strength pipeline

| Stage | Meaning |
|---|---|
| Represents | GeoMatrices supplies a coordinate/visual/operator representation. |
| Qualifies | The representation satisfies the structural assumptions required to be used in the target domain. |
| Supports | The inherited proof stack supports the relation. |
| Constrains | The target relation must satisfy closure, boundedness, transport, and observer rules. |
| Falsifies | A breach of closure, boundedness, invariance, residual tolerance, or empirical prediction rejects the mapped claim. |
| Validates internally | A claim inherits >=100% support from proven Fascicle I-II modules. |
| Corroborates externally | Empirical datasets, qualified experts, or second-method validation lift the claim toward 200%-250%. |

## Fidelity inheritance

If all supporting components have at least 100% fidelity, the Raphael-facing assembled claim may be treated as **110% internally supported** when the proof path uses more than one independent internal proof component. If a supporting proof has >110% fidelity, the downstream claim may inherit or exceed that support when the dependency is exact and non-circular.

Empirical claims still require empirical validation before 200% or 250% classification.
