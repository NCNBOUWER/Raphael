# Welcome

## EML#F

EML#F is the external EML Function alignment folder. It is used as a derivation grammar and catalogue surface for Raphael-adjacent modelling, not as a native N^3 proof module.

The source paper frames EML as a binary operator grammar for reduced nonlinear ODEs and biological dynamics. The key alignment rule is: EML supplies syntax and candidate-equation generation; semantics and empirical meaning require restricted terminals, validation rules, null models, and domain evidence.

## Start here

1. Read `Publications/EML#F overview.md`.
2. Read `INTERNAL/registers/Alignment register.md`.
3. Run `RUN/code/test_emlf_operator.py`.
4. Run `RUN/code/emlf_operator.py` for the validation report.

## Boundaries

- EML#F is separate from N^3 / GeoMatrices.
- EML#F can later host a GeoMatrices component, but N^3 does not embed EML#F.
- EML#F is not Raphael Lite.
- EML#F does not assert a new physical law from the EML operator alone.
